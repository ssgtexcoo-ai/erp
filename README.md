# SAMRUQ ERP

Единая система управления продажами, строительством и сотрудниками для Samruq Qurylys.

## Версия 1.0

### Цель проекта
Собрать в одной системе весь путь клиента от рекламы до сдачи объекта, чтобы директор получал ключевые метрики за 30 секунд.

### Ключевые модули
- Дашборд директора
- Лиды и маркетинг
- CRM сделки
- Рейтинг и мотивация сотрудников
- Объекты и этапы строительства
- Канбан и Гант
- Документы
- Уведомления
- Права доступа

### Технологический стек
- Frontend: Next.js + TypeScript + Tailwind CSS
- Backend: Supabase (PostgreSQL + Auth + Storage)
- База данных: PostgreSQL

### MVP
Через 45 дней система должна позволять:
- получать лиды
- распределять лиды
- контролировать менеджеров
- вести сделки
- создавать объекты
- управлять задачами
- работать через Канбан
- работать через Гант
- хранить документы
- показывать рейтинг сотрудников
- контролировать эффективность компании из одного окна

### Структура
- `app/` — фронтенд приложения и маршруты
- `components/` — повторно используемые UI-компоненты
- `lib/` — клиент Supabase и типы
- `db/schema.sql` — начальная схема базы данных
- `db/seed.sql` — начальные справочники и роли

### Доступные страницы
- `/` — главная
- `/login` — макет входа
- `/dashboard` — дашборд директора
- `/leads` — лиды и маркетинг
- `/deals` — CRM сделки
- `/projects` — объекты
- `/kanban` — канбан
- `/gantt` — диаграмма Ганта
- `/documents` — документы
- `/notifications` — уведомления
- `/employee` — дашборд сотрудника

### Установка и запуск
1. Установите Node.js 20+ и npm.
2. Скопируйте `.env.example` в `.env.local`.
3. Настройте `NEXT_PUBLIC_SUPABASE_URL` и `NEXT_PUBLIC_SUPABASE_ANON_KEY`.
4. Создайте проект Supabase и включите Email Auth.
5. Импортируйте `db/schema.sql` и `db/seed.sql` в базу данных.
6. Заполните реальные `auth_id` в таблице `users` после создания пользователей в Supabase Auth.
7. Запустите `npm install`.
8. Запустите `npm run dev`.

### Интеграционные проверки
- Запуск локально: `npm run test:integration`
- Эти проверки валидируют контракты модулей (схема БД, CRUD-методы сервисов, ключевые UI-действия).

### Как связать Supabase Auth с приложением
- В Supabase Auth создайте пользователей с email:
  - `director@samruq.kz`
  - `manager1@samruq.kz`
  - `manager2@samruq.kz`
  - `projectmgr@samruq.kz`
  - `accountant@samruq.kz`
  - `procurement@samruq.kz`
- В таблице `auth.users` найдите UUID каждого пользователя.
- В таблице `users` замените `auth_id` на соответствующий UUID для каждой строки.

### Что реализовано
- Полный набор страниц ERP: дашборд, лиды, CRM, объекты, канбан, диаграмма Ганта, документы, уведомления, рейтинг сотрудников
- Supabase Auth с защитой всех страниц и ролевой моделью доступа
- Редактор/создание/удаление для всех сущностей: лиды, сделки, задачи, объекты, документы
- Автоматическое распределение лидов (round-robin, load-balance) с SLA
- Диаграмма Ганта с фильтрами по статусу, ответственному, дате и поиску
- Канбан с 5 колонками и полным редактором задач
- Дашборд директора с ключевыми метриками и прогрессом этапов объектов
- GitHub Actions CI (lint, typecheck, build, интеграционные тесты)
- Telegram уведомления при завершении workflow (нужны секреты `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`)
- Авто-мерж PR с меткой `bymax-automerge`
- Ночной scheduled workflow для авто-fix lint/format
